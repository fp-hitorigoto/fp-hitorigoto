## 記事スタイルガイドライン（必ず守ること）

### ブログ記事
- **分量**：1,500字前後
- **文体**：「ですね・ですよね」を自然な流れで使う。寄り添うトーン
- **構成**：結論ファースト → 理由・背景 → 実生活への影響 → FP試験のポイント → まとめ
- **必須要素**：
  - 事実ベース（官公庁・政府HPへのリンクを入れる）
  - なぜ・どうしてという視点で実生活への影響を具体的に説明
  - 「FP試験のポイント」セクション必須
  - 試験での出題例・過去問のような問われ方も紹介
  - 学んだことが実生活でどう役立つかの視点
  - 法改正があれば今後の試験への影響にも言及

### note記事
- **文体**：同じく「ですね・ですよね」調。より語りかける感じ
- **内容**：同じテーマでも受験対策より実生活への応用を中心に
- **NG**：専門用語を多用しない。FP試験の話は不要

### 共通NG事項
- 曖昧な表現（「場合によります」だけで終わる）
- 根拠のない断定
- 金融・制度情報の不正確な記述

## 記事公開フロー（標準ワークフロー）

記事作成から公開まで、毎回この順番で進める。

### STEP 1：記事作成
- ブログ記事：`src/content/articles/記事名.md`
- note記事：`docs/note_記事名.md`（同テーマ・一般向け）
- 両方を同時に作成する

### STEP 2：素材作成
- noteカバー画像を生成（1280×670px・ネイビー系・Pillowで生成）
  - 保存先：`/Users/ohatamasanori/Downloads/note_記事名_cover.png`
- X投稿用ハッシュタグ・投稿文を作成
  - ブログURL：`https://fp-hitorigoto.github.io/fp-hitorigoto/articles/記事スラッグ`

### STEP 3：確認・push
```bash
npm run build   # エラーがないことを確認
git add src/content/articles/記事名.md
git commit -m "feat: 記事タイトルの要約"
git push origin main
# → GitHub Actionsが自動でデプロイ
```
※ **push前に必ずしゅうぞうさんに内容確認してもらう**

### STEP 4：しゅうぞうさんが手動で投稿
以下をまとめて提示してからpushする：

| 媒体 | Claudeがすること | しゅうぞうさんがすること |
|------|----------------|----------------------|
| ブログ | 記事作成・push | 確認するだけ |
| note | 本文作成・画像生成 | コピペして投稿 |
| X | 投稿文・ハッシュタグ作成 | コピペして投稿 |

### 画像生成のコツ
- 右側に大きな円（DARK2色）＋その中にシンプルなアイコン
- アイコンは幾何学形または文字（♪など）で描く。手描き風はチープに見える
- 右上・左下に装飾円（DARK1色）
- フォント：ヒラギノ角ゴシック W6（タイトル）・W3（本文）

### 注意事項
- 記事内の数字・制度情報は必ずウェブ検索で一次ソース確認
- 報道段階の情報は「〜の見込み」「一部報道によると」と明記
- 日経新聞などの有料記事はリンクせず「出典：〇〇」とテキスト明記

## note記事の共通スタイル（実例から把握）

- **冒頭**：読者の悩み・不安をセリフ形式で引用（「〜かな…」「〜ですよね」）
- **見出し**：疑問形が多い（「どういう仕組み？」「どのくらい増えるの？」「いつ聴けばいいの？」）
- **締め**：「〜してみてくださいね」「一人で抱え込まないでくださいね」など寄り添いで終わる
- **参考リンク**：末尾にまとめて記載。有料記事はリンクなしでテキスト表記
