## 現在の運用フロー（2026-07時点・確立済み）

### 毎朝の自動処理（GitHub Actions）
1. **毎朝自動記事生成**（`daily_update.yml`／`scripts/fetch_and_generate.py`・7:00 JST）
   - 官公庁RSS → キーワードフィルタ → Haiku選定 → **Haiku**で記事生成 → 自動push
   - 実際はほとんど生成されない（フィルタが厳しい・正常動作）
2. **毎朝トレンド記事生成**（`trends_update.yml`／`scripts/fetch_trends_and_generate.py`・8:00 JST）
   - Google Trends・NHK・Yahoo!ニュース → Haikuで記事化判定 → **Sonnet**でブログ記事生成 → **同時にnote下書きも生成**（`docs/note_*.md`）→ 自動push
3. push後、**GitHub Pagesが自動デプロイ**（両ワークフローの完了をトリガーに起動する設定済み）

### しゅうぞうさんとClaudeの分担（記事生成後）
しゅうぞうさんが「noteの下書きありますか」と聞く →

| 媒体 | Claudeがすること | しゅうぞうさんがすること |

| ブログ | 生成済み記事の内容チェック（カテゴリ・FP試験ポイント・事実確認） | 確認するだけ |
| note | 下書きを読み、**本文（コピペ用）とハッシュタグを別枠で**チャットに表示・タイトル案提示・カバー画像生成 | コピペして投稿 |

- 複雑なイラスト（手・人物・買い物カゴ等）はNG。文字＋シンプルな図形・数字で構成する

## 記事チェックで見るポイント（過去に実際にあった問題）

1. **事実に基づかない記述**：「FP試験でも近年出題」等の根拠不明な断定（07-05にプロスペクト理論記事で発生→修正済み）。怪しい記述はWebSearchで裏取りする

3. **同一ニュースの重複記事**：別ソース・別タイトルですり抜けることがあった（税収84兆円・日銀短観・確定拠出年金で発生→いずれか削除で対応済み）

5. **RSS見出しそのままのタイトル**：官庁発表の転載っぽい記事は要注意

## 記事スタイルガイドライン

### ブログ記事
- **文体**：「ですね・ですよね」を自然な流れで使う。寄り添うトーン
- **構成**：結論ファースト → 理由・背景 → FP試験のポイント（出題範囲内の場合）→ 実生活への影響・活用法 → まとめ
- **必須**：事実ベース。不確かな情報は「〜とされています」「要確認」と明記

### note記事
- **文体**：同じ調子でより語りかける感じ。「もしあなたが〜だったら」「実際に相談を受けることがあります」
- **内容**：実生活への応用中心。FP試験セクションは入れない
- **分量**：1200〜1800字程度

### 共通NG
- 根拠のない断定（特に金融・制度情報）
- 所属組織が特定できる表現（「金融機関の現場では」のような一般表現に）
- 記事本文へのURL・リンク記載
